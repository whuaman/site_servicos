CÓDIGO FRONT-END

Produzimos o código front-end do seu layout. Você nos envia seu layout, nós transformamos em código estático pronto para você implementar o seu código back-end.

TEMA WORDPRESS

O WordPress é um CMS versátil e muito popular. Nós podemos desenvolver seu website com WordPress, criando um theme do zero, implementando o código front-end e o integrando com o WordPress, sempre se baseando pelo seu layout.

Nosso stack de tecnologias

Você recebe o pacote com código estático e pronto para usar. Mas nós usamos um stack moderno de tecnologias para desenvolver esse código para você.

  • Middleman

    Para facilitar o build do projeto, gerenciar e produzir os arquivos estáticos, usamos o Middleman.

  • SASS

    Usamos SASS para pré-processar nosso CSS. Assim conseguimos agilizar nosso trabalho usando as vantagens que o CSS puro não tem ainda.

  • jQuery

    A biblioteca JavaScript mais popular que existe é o jQuery. Nós usamos em todos os projetos, para facilitar a manutenção futuramente. Por curiosidade, usamos Revealing Module como pattern de código.

  • Gulp

    Quando necessário, usamos Gulp para automatizar tarefas repetitivas. Geralmente o Middleman já consegue fazer o básico, as se precisamos fazer algo mais específico, preferimos usar o Gulp.

CÓDIGO RESPONSIVO

COM OU SEM BOOTSTRAP

Nós podemos desenvolver código front-end responsivo do seu layout, usando ou não o grid do Bootsrap (framework mais comum do mercado), você escolhe. É preferível que você tenha o layout mobile do seu site já desenhado. Se não tiver, sem problemas, usamos o Bootstrap para fazer os breakpoints das telas.

PRONTO PARA USAR

POUCA MANUTENÇÃO E ENTREGA RÁPIDA

Nosso código é compatível com os browsers mais modernos e super semântico. Nós podemos fazer código para browsers antigos, mas só se você precisar. Não recomendamos por vários motivos: o projeto sempre fica mais caro e complexo.

Nós entregamos esse código via GIT (nós usamos BitBucket, basta passar seu login ao final do projeto), ou se preferir, podemos enviar apenas um ZIP, com o código de desenvolvimento (normalmente feito em Middleman) e o build estático com o código de entrega.
